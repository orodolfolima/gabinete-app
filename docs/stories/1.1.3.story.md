# Story 1.1.3: Sistema de Agendamentos

## 🎯 Objetivo

Gerenciar agendamentos de visitantes com validação de conflitos, check-in/out e notificações automáticas.

## ✅ Aceitação

### Endpoints
- POST /api/agendamentos (criar)
- GET /api/agendamentos (listar)
- PUT /api/agendamentos/:id (atualizar)
- DELETE /api/agendamentos/:id (cancelar)
- POST /api/agendamentos/:id/check-in
- POST /api/agendamentos/:id/check-out

### Validações
- Conflito de horário (horário sobreposto)
- Visitante existe
- Data não é no passado
- Duração válida

### Automações
- SMS confirmação 24h antes
- SMS lembrete 2h antes
- Cancelamento automático após 1h sem check-in
- Email com resumo pós-visita

## 📝 Schema

```prisma
model Agendamento {
  id              String
  visitante_id    String
  visitante       Visitante @relation(...)
  data_hora       DateTime
  duracao_min     Int
  tipo            String
  assunto         String
  status          String // agendado, confirmado, realizado, cancelado
  responsavel     String
  check_in        DateTime?
  check_out       DateTime?
  notificacao_sent Boolean @default(false)
  created_at      DateTime
  updated_at      DateTime
}
```

## 🧪 Testes

- Detecta conflito de horário
- Valida datas passadas
- Check-in/out sequence correto
- Notificações disparadas
