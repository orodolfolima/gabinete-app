# Story 1.1.1: Modelo de Dados + Migrations (Visitantes)

## 📋 Contexto

Primeira story do Módulo 1. Cria schema PostgreSQL para visitantes com relacionamentos, índices e RLS.

## 🎯 Objetivo

Estabelecer modelo de dados robusto que suporte:
- Cadastro completo de visitantes
- Histórico de interações
- RLS para isolamento de dados
- Performance com índices

## ✅ Acceptance Criteria

### Tabela `visitantes`
```sql
CREATE TABLE visitantes (
  id TEXT PRIMARY KEY,
  cpf TEXT UNIQUE NOT NULL,
  rg TEXT,
  nome TEXT NOT NULL,
  email TEXT,
  telefone TEXT,
  whatsapp TEXT,
  foto_url TEXT,
  categoria TEXT,
  deleted_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_visitantes_cpf ON visitantes(cpf);
CREATE INDEX idx_visitantes_email ON visitantes(email);
CREATE INDEX idx_visitantes_telefone ON visitantes(telefone);
CREATE INDEX idx_visitantes_categoria ON visitantes(categoria);
```

### Tabela `endereco`
```sql
CREATE TABLE endereco (
  id TEXT PRIMARY KEY,
  visitante_id TEXT UNIQUE NOT NULL REFERENCES visitantes(id) ON DELETE CASCADE,
  cep TEXT NOT NULL,
  logradouro TEXT NOT NULL,
  numero TEXT NOT NULL,
  complemento TEXT,
  bairro TEXT NOT NULL,
  cidade TEXT NOT NULL,
  estado TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### Tabela `interacao`
```sql
CREATE TABLE interacao (
  id TEXT PRIMARY KEY,
  visitante_id TEXT NOT NULL REFERENCES visitantes(id) ON DELETE CASCADE,
  tipo TEXT,
  descricao TEXT,
  data TIMESTAMP DEFAULT NOW(),
  usuario_id TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_interacao_visitante ON interacao(visitante_id);
CREATE INDEX idx_interacao_data ON interacao(data);
```

### RLS Policy
```sql
ALTER TABLE visitantes ENABLE ROW LEVEL SECURITY;

CREATE POLICY visitantes_isolacao
  ON visitantes
  FOR SELECT
  USING (setor_responsavel = current_setting('app.user_setor'));
```

### Soft Delete Trigger
```sql
CREATE TRIGGER visitantes_soft_delete
BEFORE DELETE ON visitantes
FOR EACH ROW
EXECUTE FUNCTION soft_delete_trigger();
```

## 📝 Prisma Schema

```prisma
model Visitante {
  id         String   @id @default(cuid())
  cpf        String   @unique
  rg         String?
  nome       String
  email      String?
  telefone   String?
  whatsapp   String?
  foto_url   String?
  categoria  String?

  endereco   Endereco?
  interacoes Interacao[]

  deleted_at DateTime?
  created_at DateTime @default(now())
  updated_at DateTime @updatedAt

  @@index([cpf])
  @@index([email])
  @@index([telefone])
}

model Endereco {
  id         String   @id @default(cuid())
  visitante_id String  @unique
  visitante  Visitante @relation(fields: [visitante_id], references: [id], onDelete: Cascade)

  cep        String
  logradouro String
  numero     String
  complemento String?
  bairro     String
  cidade     String
  estado     String

  created_at DateTime @default(now())
  updated_at DateTime @updatedAt
}

model Interacao {
  id         String   @id @default(cuid())
  visitante_id String
  visitante  Visitante @relation(fields: [visitante_id], references: [id], onDelete: Cascade)

  tipo       String?
  descricao  String?
  data       DateTime @default(now())
  usuario_id String?

  created_at DateTime @default(now())

  @@index([visitante_id])
  @@index([data])
}
```

## 🧪 Testes

### Teste 1: Constraint CPF
```typescript
test('CPF deve ser único', async () => {
  const v1 = await prisma.visitante.create({
    data: { cpf: '12345678901', nome: 'João' }
  });

  expect(async () => {
    await prisma.visitante.create({
      data: { cpf: '12345678901', nome: 'Maria' }
    });
  }).toThrow();
});
```

### Teste 2: Cascade Delete
```typescript
test('Deletar visitante remove endereço', async () => {
  const v = await prisma.visitante.create({
    data: {
      cpf: '12345678901',
      nome: 'João',
      endereco: { create: { cep: '01310100', ... } }
    },
    include: { endereco: true }
  });

  await prisma.visitante.delete({ where: { id: v.id } });

  const endereco = await prisma.endereco.findUnique({
    where: { visitante_id: v.id }
  });
  expect(endereco).toBeNull();
});
```

### Teste 3: Performance
```typescript
test('Índice em CPF melhora query', async () => {
  const start = Date.now();
  await prisma.visitante.findUnique({ where: { cpf: '12345678901' } });
  const duration = Date.now() - start;

  expect(duration).toBeLessThan(10); // < 10ms
});
```
