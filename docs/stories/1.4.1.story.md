# Story 1.4.1: Templates de Mensagens

## 📋 Contexto

Sistema de templates reutilizáveis para automação de comunicação via SMS, WhatsApp e Email. Suporte a variáveis dinâmicas e histórico de versões.

## 🎯 Objetivo

Permitir que operadores criem e gerenciem templates de mensagens com:
- Suporte multi-canal (SMS, WhatsApp, Email)
- Variáveis dinâmicas (nome, data, etc)
- Preview em tempo real
- Histórico de mudanças
- Testes antes de publicar

## ✅ Acceptance Criteria

### AC1: Modelo de Dados
- [ ] Tabela `templates` com campos: id, titulo, conteudo, canal, variáveis, ativo, versão
- [ ] Tabela `template_versoes` para histórico de mudanças
- [ ] Índices em canal, ativo, criado_em

### AC2: API CRUD
- [ ] `POST /api/templates` — Criar template
- [ ] `GET /api/templates` — Listar (com filtros: canal, ativo)
- [ ] `GET /api/templates/:id` — Detalhe
- [ ] `PUT /api/templates/:id` — Atualizar (cria nova versão)
- [ ] `DELETE /api/templates/:id` — Deletar (soft delete)
- [ ] `GET /api/templates/:id/versoes` — Histórico

### AC3: Variáveis Dinâmicas
- [ ] Suporte para placeholders: {nome}, {data}, {hora}, {categoria}, {parlamentar}
- [ ] Validação: variáveis usadas existem
- [ ] Preview renderiza variáveis corretamente
- [ ] Template mostra lista de variáveis disponíveis

### AC4: Editor Visual
- [ ] Componente textarea com markdown support
- [ ] Preview live ao lado do editor
- [ ] Botão "Adicionar variável" com dropdown
- [ ] Validação de comprimento por canal:
  - SMS: máx 160 caracteres (ou múltiplo de 160)
  - WhatsApp: máx 4096
  - Email: sem limite

### AC5: Templates Pré-configurados
- [ ] CONFIRMACAO_AGENDAMENTO
- [ ] LEMBRETE_24H
- [ ] LEMBRETE_2H
- [ ] FOLLOW_UP_POS_VISITA
- [ ] PESQUISA_SATISFACAO
- [ ] AVISOS_GERAIS

### AC6: Testes
- [ ] Endpoint para enviar template teste (SMS/WhatsApp/Email)
- [ ] Log de testes disparados
- [ ] Feedback de sucesso/erro
