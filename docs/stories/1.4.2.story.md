# Story 1.4.2: Integração n8n

## 📋 Contexto

Configurar n8n (workflow automation platform) para automatizar envios de mensagens baseados em triggers da aplicação (novo agendamento, demanda criada, aniversário, etc).

## 🎯 Objetivo

Estabelecer pipeline de automação que:
1. Dispara workflows quando eventos ocorrem (via webhooks)
2. Renderiza templates de mensagens (integração com 1.4.1)
3. Envia mensagens via WhatsApp/SMS/Email
4. Registra falhas e tenta novamente
5. Notifica administrador em caso de erro

## ✅ Acceptance Criteria

### AC1: Setup n8n Self-Hosted
- [ ] Container n8n adicionado ao docker-compose.yml
- [ ] Volumes para persistência (workflows, execuções)
- [ ] Health check configurado
- [ ] Restart policy: always
- [ ] n8n acessível em http://localhost:5678

### AC2: Webhooks da API
- [ ] POST /api/webhooks/n8n/:event — dispara workflow
- [ ] Payloads validados
- [ ] Retry automático (3x) em caso de falha
- [ ] Logging de chamadas

### AC3: Workflows Implementados (5)
- [ ] Workflow 1: Confirmação de Agendamento (24h antes)
- [ ] Workflow 2: Lembrete 2h antes
- [ ] Workflow 3: Follow-up pós-visita
- [ ] Workflow 4: Notificação de demanda criada
- [ ] Workflow 5: Aviso genérico (admin trigger)

### AC4: Estrutura de Cada Workflow
```
Trigger (webhook) → Filter → Template API → Enviar (WhatsApp/SMS/Email) → Log
```

### AC5: Notificações de Erro
- [ ] Slack webhook integrado (opcional)
- [ ] Email em caso de falha crítica
- [ ] Dead letter queue para reprocessamento

### AC6: Documentação
- [ ] Arquivo com diagrama de workflows
- [ ] Como adicionar novo workflow
- [ ] Credenciais necessárias (documentadas em .env.example)
